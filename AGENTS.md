# AI Diagnostics Micro App - Development Guide

## Overview
This project is an automated B2C "AI Diagnostics/Fortune Telling" application. The goal is to generate $3k/month in passive income by selling personalized PDF reports generated by an LLM (e.g., OpenAI GPT-4o) based on user input.

**Target Flow**:
1. User sees a TikTok/Instagram ad and clicks the link in bio.
2. User lands on a clean, modern Next.js landing page.
3. User fills out a multi-step form (e.g., name, birthdate, specific concerns).
4. User pays via Stripe Checkout ($15 - $20 per report).
5. Stripe Webhook triggers a backend API route.
6. The backend sends a meticulously crafted prompt with the user's data to the OpenAI API.
7. OpenAI generates a detailed, 3,000+ word personalized markdown report.
8. The backend converts the markdown to a beautifully formatted PDF.
9. The backend automatically emails the PDF to the user (via Resend/SendGrid).

## Tech Stack
- **Frontend**: Next.js (App Router), React, Tailwind CSS.
- **Backend**: Next.js API Routes (Serverless).
- **Database (Optional/Minimal)**: Supabase (PostgreSQL) - mainly for storing order status and email logs.
- **Payments**: Stripe Checkout & Webhooks.
- **AI**: OpenAI API (gpt-4o or gpt-4o-mini).
- **Email**: Resend.
- **PDF Generation**: `react-pdf` or Puppeteer/Playwright (headless browser approach for perfect styling).

## System Prompts (AI Coding Assistant Rules)
If you are an AI assistant (like Cursor, Bolt, or Lovable) reading this file to help build the app, follow these strict rules:

1. **Aesthetics First**: The UI must look incredibly premium. Use smooth micro-animations (Framer Motion), glassmorphism, and a high-converting, trustworthy layout. Do not build a "boring" SaaS dashboard; build an intuitive, mobile-first consumer experience.
2. **Server Actions vs API Routes**: Use Next.js Server Actions for form submissions. Use standard API Routes (`/api/stripe-webhook`) explicitly for the Stripe webhook to avoid edge runtime limitations when generating PDFs.
3. **Robust Webhooks**: The Stripe webhook must be deeply robust. It must:
   - Verify the Stripe signature.
   - Return a `200 OK` to Stripe *immediately* before starting the slow LLM generation, or use a background queue (like Inngest/Upstash QStash) to prevent Stripe webhook timeouts (LLM generation + PDF conversion takes > 10 seconds).
4. **The "Wow" Factor in the Content**: The system prompt sent to OpenAI must enforce a highly empathetic, detailed, and visually structured output. The generated content is the actual product being sold. It must be worth the money.
